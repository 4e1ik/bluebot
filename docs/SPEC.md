# Спецификация blubot

## Назначение

Telegram-бот каталога товаров на продажу с белым списком пользователей и ручным подтверждением брони админом.

## Роли

### Админ (`ADMIN_ID` из `.env`)
- Добавляет товары: фото → название → цена
- Управляет белым списком: `/adduser`, `/removeuser`, `/users`
- Просматривает корзины пользователей в «Ожидают подтверждения»
- Подтверждает/отклоняет всю корзину или отдельный товар

### Пользователь (в whitelist)
- Просматривает каталог
- Бронирует товары
- Видит «Мои брони» (своя корзина)
- Отменяет свои pending-брони
- После бронирования получает инструкцию написать админу `@ADMIN_USERNAME`

## Статусы товара

| Статус | Виден в каталоге | Отображение |
|--------|------------------|-------------|
| `available` | Да | Кнопка «Забронировать» |
| `pending` | Да | «Забронирован» или «Забронировано вами» |
| `hidden` | Нет | Продан/забран после подтверждения админом |

## Статусы брони

| Статус | Описание |
|--------|----------|
| `pending` | Активная бронь, до 12 часов |
| `confirmed` | Админ подтвердил |
| `rejected` | Админ отклонил |
| `cancelled` | Пользователь отменил |
| `expired` | Прошло 12ч без подтверждения |

## Корзина

Все `pending`-брони одного пользователя = его корзина. Отдельной таблицы нет.

Админ видит:
1. Список пользователей с количеством товаров и суммой
2. Детали корзины конкретного пользователя
3. Действия: подтвердить/отклонить всё или поштучно

## Срок брони

- TTL: 12 часов (`BOOKING_TTL_HOURS`)
- Фоновая задача каждые 5 мин снимает просроченные брони
- Товар возвращается в `available`
- Пользователь получает уведомление

## Инструкция после брони

```
Товар забронирован на 12 часов.
Когда закончите выбор товаров — напишите админу @ADMIN_USERNAME для согласования.
```

## Ограничения

- Пользователь обязан иметь `@username` в Telegram
- Один товар — одна активная бронь
- Админу уведомления о новых бронях не отправляются
- Нет редактирования товара после добавления (v1)
- Проданные товары скрыты из UI, данные остаются в БД
