# Спецификация blubot

## Назначение

Telegram-бот каталога товаров на продажу с белым списком пользователей и ручным подтверждением брони админом.

## Роли

### Суперпользователь (`SUPERUSER_ID` / `SUPERUSER_USERNAME` из `.env`)
- Всё, что может обычный админ
- Управляет списком админов: кнопка «Админы» → добавить/удалить по `@username`
- Самого супера нельзя удалить через бота
- Уведомления о заказах **не** получает

### Админ (таблица `admins`)
- Добавляет/удаляет товары
- Управляет белым списком пользователей
- Получает уведомление при **«Оформить заказ»**
- Подтверждает/отклоняет заказы в «Ожидают подтверждения»

### Пользователь (в whitelist)
- Каталог (список → фото по клику)
- Бронирует товары в корзину (TTL 20 мин)
- «Мои брони»: оформить заказ / очистить / отменить товар
- После оформления: «Ожидайте, скоро с вами свяжутся»
- Кнопка «Помощь» с инструкцией

## Корзина и заказ

| Статус брони | Смысл |
|--------------|-------|
| `pending` | В корзине, ещё не оформлен (20 мин TTL) |
| `submitted` | Оформлен, ждёт админа |
| `confirmed` / `rejected` / `cancelled` / `expired` | Финальные |

- Автоотмена только для `pending` через 20 минут
- Админ видит в «Ожидают» только `submitted`

## Каталог

- Список inline-кнопок (длинные названия обрезаются)
- Фото — только на карточке после клика

## Protect content

Все сообщения бота с `protect_content=True` — запрет пересылки.

## Ограничения

- Админ должен хотя бы раз написать боту (`/start`), иначе `user_id` неизвестен и уведомления не дойдут
- Полный запрет скриншотов на всех клиентах невозможен
